# QA Concept

## About the Project

Our team combines occasional in-person meetings with continuous communication through Discord and WhatsApp. Since we stay in constant contact via WhatsApp throughout the week — sharing updates, asking questions, and keeping each other informed in real time — formal weekly meetings are rarely necessary. When we do meet, either in person or online, we use the time to review progress, address open issues, and plan the next steps. Task distribution follows a flexible, milestone-based approach, ensuring responsibilities are aligned with the current state of the project. Completed work is regularly presented during these sessions to gather feedback and improve overall code quality.

## Coding Standards

To ensure consistency across the codebase, our team follows agreed-upon coding standards based on Java conventions. Our IDEs are configured to automatically format code on save, reducing style inconsistencies and keeping the code clean without extra effort.

## Code Review

Documentation is treated as a mandatory part of development. We write JavaDoc for all classes and methods, and maintain a central README file that serves as the primary reference point for the project structure, setup, and usage.

## Version Control

We use GitHub for version control, following a structured branching strategy to keep the main branch stable and all development traceable. The main branch only contains tested, production-ready code. For each new feature or task, team members create their own branch based on the latest version of main, committing changes regularly with clear and descriptive messages.
